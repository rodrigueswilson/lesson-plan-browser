# MockFlow Free Tier Usage Guide
## Strategic Planning for Browser Module & Lesson Mode

**Date**: 2025-01-27  
**Tool**: MockFlow Free Tier  
**Goal**: Maximize efficiency within free tier limitations

---

## Free Tier Limitations

### What You Get:
- ✅ **1 Wireframe Project** (3 pages maximum)
- ✅ **3 IdeaBoard Projects** (for brainstorming)
- ✅ **1 SiteMap Project** (for navigation structure)
- ✅ **100 AI Credits** (for AI-powered features)
- ✅ **Public Sharing** (share via links)
- ✅ **All UI Packs & Templates** (access to all design elements)

### What's Restricted:
- ❌ Multiple wireframe projects
- ❌ More than 3 pages per wireframe project
- ❌ Private collaboration
- ❌ Advanced export formats (some limitations)

**Page 2: Lesson Mode - Simplified MVP** (Priority #2)

Since you get **3 IdeaBoard projects**, use them for:
- **IdeaBoard 1**: Component inventory and UI patterns

## Efficient AI Credit Usage (100 Credits)

### High-Value Uses (Use Credits Here):
1. **Text-to-Wireframe** (10-20 credits)
   - Generate initial wireframe from your requirements
   - Use for: "Create a wireframe for a lesson plan browser with current lesson display, schedule navigation, and filtering"

2. **AI-Assisted Layout** (5-10 credits per screen)
   - Quick layout suggestions
   - Component placement recommendations

3. **Component Generation** (2-5 credits per component)
   - Generate specific UI components
   - Example: "Create a timer display component with progress bar"

### Low-Value Uses (Avoid):
- ❌ Multiple iterations of the same screen
- ❌ Complex animations (not needed for wireframes)
- ❌ High-fidelity design details (wireframes should be low-fidelity)

### Credit Conservation Strategy:
- **Start with manual wireframing** using templates/UI packs
- **Use AI credits only for**:
  - Initial wireframe generation (one-time)
  - Complex component suggestions
  - Layout optimization (if needed)

## Recommended Workflow

### Step 1: SiteMap First (Use SiteMap Project)

**Why Start Here:**
- SiteMap is separate from wireframe project (doesn't count toward 3 pages)
- Helps visualize navigation structure
- Can be exported/shared easily

**What to Create:**
```
App Root
├── Browser Module
│   ├── Dashboard (Current Lesson)
│   ├── Schedule View
│   ├── Lesson Plan Browser
│   └── Filters
└── Lesson Mode
    ├── Entry Screen
    ├── Active Lesson View
    └── Step Navigation
```

**How to Share:**
- Export as image or PDF
- Share public link
- Copy/paste structure as text

**Focus**: Core Browser Module screens

**Screens to Include:**
1. **Current Lesson Dashboard** (main screen)

   - Quick actions (View Plan, Enter Lesson Mode)

2. **Schedule View** (secondary screen - can combine with dashboard)
   - Weekly calendar
   - Day view
   - Filter controls

**AI Credit Usage**: 
- Use 15-20 credits for initial wireframe generation
- Prompt: "Create a wireframe for a lesson plan browser dashboard showing current lesson, time remaining, and navigation controls"

- Export as PNG/PDF
- Share public link
- Take screenshots of key sections

### Step 3: Wireframe Project - Page 2 (Lesson Mode MVP)

**Focus**: Simplified Lesson Mode (v1 MVP)

**Screens to Include:**

2. **Active Lesson View**
   - Timer display (manual start/stop)

**AI Credit Usage**:
- Use 15-20 credits for initial wireframe
- Prompt: "Create a wireframe for a lesson mode screen with manual timer, step navigation, and large text display for sentence frames"

**Export Strategy**:
- Export as PNG/PDF

3. **Note** any specific design decisions

```markdown
**Layout Description**:
- [Component 1]: [Location and purpose]

- [Component 3]: [Location and purpose]

**Key Interactions**:
- [Interaction 1]: [What happens]
- [Interaction 2]: [What happens]

**Design Decisions**:
- [Decision 1]: [Rationale]
- [Decision 2]: [Rationale]

**Screenshot/Image**: [Attach or link]
```

### Example:

```markdown
## Page 1: Browser Module - Current Lesson Dashboard

**Purpose**: Quick access to current lesson with time-based detection

**Layout Description**:
- Header: App title and navigation
- Current Lesson Card (center): Shows subject, grade, time range, time remaining
- Quick Actions (below card): "View Full Plan" and "Enter Lesson Mode" buttons
- Navigation (bottom): Previous/Next lesson cards
- Week Overview (sidebar): Calendar showing lessons per day

**Key Interactions**:
- Click "View Full Plan": Opens lesson plan detail view
- Click "Enter Lesson Mode": Transitions to Lesson Mode
- Click Previous/Next: Navigate to adjacent lessons

**Design Decisions**:
- Current lesson card is prominent (large, centered)
- Timer countdown is visually distinct (large, colored)
- Quick actions are easily accessible (primary buttons)

**Screenshot**: [Attach image]
```

## Maximizing Free Tier: Pro Tips

### 1. Use Templates & UI Packs
- **Don't start from scratch** - use MockFlow's templates
- **Leverage UI packs** - they're all available in free tier
- **Save AI credits** by using pre-built components

### 2. Combine Related Screens
- **One page can show multiple states** (e.g., show filter panel open/closed)
- **Use annotations** to explain different views
- **Don't create separate pages** for minor variations

### 3. Use IdeaBoards for Planning
- **IdeaBoard 1**: Component inventory
- **IdeaBoard 2**: User flow diagrams
- **IdeaBoard 3**: Platform considerations
- **These don't count** toward wireframe pages!

### 4. Export Early and Often
- **Export wireframes** as you complete them
- **Save versions** locally (free tier may have export limits)
- **Document decisions** in text alongside wireframes

### 5. Prioritize Based on Implementation Order
- **Page 1**: Browser Module (Phase 1-2 priority)
- **Page 2**: Lesson Mode MVP (Phase 3 priority)
- **Page 3**: Navigation/SiteMap (can use IdeaBoard instead)
