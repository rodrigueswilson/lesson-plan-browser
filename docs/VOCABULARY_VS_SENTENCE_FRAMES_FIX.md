# Vocabulary vs Sentence Frames Consistency Fix

## Problem Summary

After implementing the initial fix to make Vocabulary / Cognate Awareness sections mandatory, the issue persisted:
- **25 "Sentence Frames / Stems / Questions:" sections** (correct - 5 days × 5 slots = 25)
- **Only 12 "Vocabulary / Cognate Awareness:" sections** (should be 25)

This indicated that while Sentence Frames was always being created, Vocabulary sections were still missing from some lessons.

## Root Cause Analysis

### Key Difference Found

**Sentence Frames Section** (line 3062):
```python
if day_sentence_frames:
    # Create sentence frames step...
```
- **Conditional creation** - only creates if `day_sentence_frames` exists
- **BUT** - `sentence_frames` was always being generated by the LLM, so this condition always passed
- Result: Always created (25/25)

**Vocabulary Section** (line 2929):
```python
should_create_vocab_step = True
if should_create_vocab_step:
    # Create vocabulary step...
```
- **Should always create** - `should_create_vocab_step` is always True
- **BUT** - `vocabulary_cognates` was sometimes missing/empty, and the code might not have been executing properly
- Result: Only created when vocabulary_cognates was present (12/25)

### Additional Differences

1. **generated_steps handling**: 
   - Vocabulary: Always added to `generated_steps` after database save
   - Sentence Frames: Only added to `generated_steps` if table was missing

2. **Placeholder content**:
   - Vocabulary: Had placeholder when empty
   - Sentence Frames: Would fail silently if empty (though this didn't happen in practice)

## Solutions Implemented

### 1. Made Sentence Frames Mandatory (backend/api.py)

**Change**: Made Sentence Frames section always create, matching Vocabulary behavior.

**Before**:
```python
if day_sentence_frames:
    # Only create if sentence_frames exists
```

**After**:
```python
should_create_frames_step = True
if not day_sentence_frames or (isinstance(day_sentence_frames, list) and len(day_sentence_frames) == 0):
    # Log warning but still create
    logger.warning(...)

if should_create_frames_step:
    # Always create the section
    # Use placeholder content if sentence_frames is empty
```

**Benefits**:
- Ensures both sections are created consistently
- Same pattern as Vocabulary section
- Provides placeholder when data is missing

### 2. Added Placeholder Content for Sentence Frames

**Change**: Added placeholder content when `sentence_frames` is missing, matching Vocabulary behavior.

**Before**:
```python
if day_sentence_frames:
    frames_display_parts.append("\nReference Frames:")
    # ... add frames
```

**After**:
```python
if day_sentence_frames and isinstance(day_sentence_frames, list) and len(day_sentence_frames) > 0:
    frames_display_parts.append("\nReference Frames:")
    # ... add frames
else:
    # Placeholder content when sentence_frames is missing
    frames_display_parts.append(
        "Sentence Frames / Stems / Questions: "
        "Sentence frames will be added based on lesson objectives and content. "
        "Please ensure sentence_frames is included in the lesson plan JSON."
    )
```

### 3. Consistent generated_steps Handling

**Change**: Made Sentence Frames always add to `generated_steps` after database save, matching Vocabulary.

**Before**:
```python
try:
    created_id = db_for_plan.create_lesson_step(frames_step)
    # Only logged, didn't add to generated_steps
except:
    if is_table_missing:
        generated_steps.append(...)  # Only if table missing
```

**After**:
```python
try:
    created_id = db_for_plan.create_lesson_step(frames_step)
    # Always add to generated_steps for consistency
    frames_step_with_timestamps = {...}
    generated_steps.append(LessonStep(**frames_step_with_timestamps))
except:
    if is_table_missing:
        generated_steps.append(...)  # Fallback if table missing
```

## Code Structure After Fix

Both sections now follow the same pattern:

1. **Always create** - Both use `should_create_*_step = True`
2. **Log warnings** - Both log when data is missing
3. **Placeholder content** - Both show placeholder when data is empty
4. **Consistent saving** - Both always add to `generated_steps`
5. **Same error handling** - Both handle table missing errors the same way

## Expected Behavior After Fix

1. **Both sections will ALWAYS appear** in all lesson plans (25/25 for both)
2. **If data is missing**, both sections will display placeholder messages
3. **Consistent behavior** - Both sections are created using the same logic pattern
4. **Better debugging** - Warnings logged when data is missing for either section

## Related Files Modified

- `backend/api.py` - Made Sentence Frames mandatory and consistent with Vocabulary

## Key Takeaway

The issue was that **Sentence Frames appeared to work** because the LLM was always generating `sentence_frames` data, but **Vocabulary was failing** because `vocabulary_cognates` was sometimes missing. By making both sections **mandatory and using the same creation pattern**, we ensure consistency regardless of whether the LLM generates the data or not.
