### Phase 2: Parse `lesson_json` Field (Priority: HIGH)

**Problem**: `lesson_json` is stored but not parsed/used.

**Current State**:
- `WeeklyPlan` model has `lessonJson: String?` field
- Diagnostic screen shows `lesson_json` exists but doesn't parse it
- UI components don't use `lesson_json` data

**Solution**:
1. Create `LessonJsonParser` utility to parse JSON structure
2. Extract data from `lesson_json`:
   - `days` object (Monday-Friday)
   - Each day contains slots (1, 2, etc.)
   - Each slot contains:
     - `objectives` (content + language)
     - `vocabulary` / `vocabulary_cognates`
     - `sentence_frames`
     - `materials_needed`
     - `instruction_steps` (if not in `lesson_steps` table)
3. Update `LessonDetailView` to use parsed `lesson_json`
4. Update `LessonModeScreen` to use parsed `lesson_json` for steps

**JSON Structure** (from PC app code):
```typescript
{
  days: {
    "Monday": {
      "1": { // slot number
        objectives: { content: "...", language: "..." },
        vocabulary_cognates: [...],
        sentence_frames: [...],
        materials_needed: [...],
        instruction_steps: [...]
      },
      "2": { ... }
    },
    "Tuesday": { ... },
    ...
  }
}
```

**Files to Create/Update**:
- `android/app/src/main/java/com/bilingual/lessonplanner/utils/LessonJsonParser.kt` (NEW)
- `android/app/src/main/java/com/bilingual/lessonplanner/ui/browser/lesson/LessonDetailView.kt` (UPDATE)
- `android/app/src/main/java/com/bilingual/lessonplanner/ui/lessonmode/LessonModeScreen.kt` (UPDATE)
